OAuth 2.0 Authorization Framework

Overview

OAuth 2.0 is the industry-standard protocol for authorization. It focuses on client developer simplicity while providing specific authorization flows for web applications, desktop applications, mobile phones, and living room devices.

Implementation Details

  • Roles: Resource Owner, Client, Authorization Server, Resource Server
  • Tokens: Access Tokens are issued to third-party clients by an authorization server with the approval of the resource owner.
  • Security: Relies on HTTPS for all communication. Bearer tokens are the most common type.
  • Flows: Authorization Code, Implicit, Resource Owner Password Credentials, and Client Credentials grant types.

Use Cases

  • Third-party application access (e.g., "Login with Google")
  • Securing APIs
  • Single Sign-On (SSO) across multiple applications
  • Providing scoped access to resources
