JWT (JSON Web Token) Authentication

Overview

JWT is a compact, URL-safe means of representing claims to be transferred between two parties. It's commonly used for authentication and information exchange in web applications.

Token Structure

A JWT consists of three parts separated by dots:

  • Header: Contains token type and signing algorithm
  • Payload: Contains claims (user data, expiration, etc.)
  • Signature: Ensures token hasn't been tampered with

Implementation Details

  • Format: Authorization: Bearer
  • Stateless: No server-side session storage
  • Self-contained: All user info encoded in token
  • Expiration: Tokens have built-in expiration time

Advantages

  • Stateless and scalable
  • Cross-domain authentication
  • Mobile app friendly
  • Decentralized verification

Use Cases

  • Single Page Applications (SPAs)
  • Mobile applications
  • Microservices authentication
  • API authentication

Testing

You can test this endpoint using curl:

curl -H "Authorization: Bearer " http://localhost:8080/jwt/home
